0. このモデルの正体（最初に結論）

このモデルは：

「因果を数値で予測するモデル」ではなく
「因果が“立ち上がる条件”と“壊れる条件”を分離して観測するモデル」

です。

そのために
符号・位相・モードが必要でした。

1. 元々の構造（復元）
(1) 基本状態：Sᵢⱼ（因果状態テンソル）

Sᵢⱼ は
「i に do したとき j がどう“応答しようとするか”」

値そのものは意味を持たない

符号と位相だけが重要

符号

Sᵢⱼ > 0：活性因果（促進・支持）

Sᵢⱼ < 0：抑制因果（排他・破壊）

👉
Excel の例で言えば：

A列 → B列：正の因果

A列 → 無関係セル：ほぼ 0

A列 → 別ブロックの見出し：負の因果（意味破壊）

(2) 複素数化：Sᵢⱼ ∈ ℂ

ここが最重要ポイントです。

因果は「強さ」ではなく
「タイミングと持続性」を持つ

そのために：

$$ S_{ij}​=r_{ij}​⋅e^{iθ_{ij}} $$
​

2. モード選択的フィルタリング（Mode-selective Filtering）

あなたが確定させた核心です。

モード k の導入  
​​
$$ S_{ij}=\sum_{k}{S_{ij}}^{(k)} $$

k=0：順相・伝搬モード 位相 ≈ 0  即時に反応 減衰が弱い
学習しない（構造反射）意味
表の見出し
セル結合
配置による代表性
👉
A1:A5 = 東京都
→ B1:B5 は即時に従属

k=1：遅延・慣性モード

位相 ≈ π/2

遅れて効く

自己ループ優先

学習可能

意味

文脈

書き手の癖

過去の修正履歴（仮想）

👉
「この表では、都道府県は縦にまとめる」という
人間の無意識ルール

3. 因果慣性 と 自己慣性 の分離

ここが Simulinkと決定的に違う点です。

因果慣性（causal inertia）

Sᵢⱼ が他ノードへ影響を及ぼし続ける性質

構造由来

モード k=0 が支配的

例：

A列を書き換えると B列が壊れる

自己慣性（self inertia）

Sᵢᵢ の自己保持

表の「意図」が壊れにくい性質

モード k=1 が支配的

例：

「これは都道府県ブロックだ」という理解が簡単には消えない

4. attention の本来の役割（誤解の修正）

attention は：

類似度計算ではない
距離でもない
モードを選択するスイッチ

です。

attention がしていたこと

いまは k=0 を見るべきか？

それとも k=1 を見るべきか？

どの Sᵢⱼ を「今この step で有効化」するか？

👉
attention = 因果の位相整合器

5. step（時間発展）の意味（再定義）

1 step は：

位相が揃う／ずれる過程

であり、数値予測ではありません。

整合している → 収束

位相がずれる → 応答が振動

振動が減衰しない → 破壊的介入

これを CII / CII′ で 比較する。
